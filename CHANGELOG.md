## 📋 Versioning-Standard

### Format: vX.Y.Z

- **X (Major):** Große Meilensteine (neue Kriterien erfüllt, Phase abgeschlossen)
- **Y (Minor):** Features und Bugfixes (Kriterium-Verbesserungen)
- **Z (Patch):** Kleinere Korrektionen (Typos, CSS Fine-Tuning)

### Branch-Tags

Alle Releases werden als Git-Tags markiert:

```bash
git tag -a v1.0.0 -m "Version 1.0.0 – Verteidigungsstand"
git push origin v1.0.0
```

### Commit-Prefix-Konvention

- `[FEAT-Xy]` – Neue Features (z.B. `[FEAT-D1]` für Galerie-Feature)
- `[FIX-Xy]` – Bugfixes für Kriterien (z.B. `[FIX-I1]` für @media Fixes)
- `[DOCS]` – Dokumentations-Updates
- `[REFACTOR]` – Code-Verbesserungen ohne neue Features
- `[INIT]` – Initialisierung / erste Commits
- `[RELEASE]` – Release-Vorbereitung und Versioning
